# クラスタ適合チェック

大喜利の回答群を受け取り、ユーザークラスタ別の嗜好特徴にどれだけ合うかを分析する。

## 入力

お題と、それに対する回答群（3〜20個程度）を受け取る。

## 原則

**「面白いか」を判定するな。「どのクラスタの嗜好特徴に寄っているか」だけを判定せよ。**

このスキルの点数は、人間の笑いの絶対点ではない。回答の表層・構造特徴が、文献由来のクラスタ別嗜好特徴にどれだけ一致するかを示す開発用シグナルである。

ペルソナを演じて採点するな。クラスタを人格化すると、LLMの想像した読者像に引っ張られて評価がぶれる。必ず下の特徴表に基づいて採点し、根拠を特徴名で書け。

## 特徴抽出

各回答について、目で確認できる特徴だけを取る。迷う特徴は「弱」として扱うか、採点しない。

- 括弧の使用: `（ ）`、`「 」`、補足括弧などが効果の中心にある
- 対話形式: 誰かの発話、返答、会話の形になっている
- 複数文: 句点や明確な文の切れ目が複数ある
- 短い長さ比率: 極端に短く、情報が少ない
- 自虐ネタ: 話者や回答内主体が自分を下げる
- シュールな無意味さ: 意味の接続より不可解さで成立している
- 形容詞で終わる: 最後が形容詞・形容動詞的な評価で終わる
- 擬人化: 物や概念が人間のように振る舞う
- プロンプト内の固有名詞: お題に含まれる固有名詞を核にしている
- プロンプト内の動詞: お題に含まれる動詞を核にしている
- 三点リーダーで終わる: `...` または `…` で終わる
- ミニストーリー: 短い中に前後関係や展開がある
- 疑問符で終わる: `?` または `？` で終わる
- 高いスペース比率: 空白が間や見た目の効果の中心にある
- スラング: ネット語、若者語、砕けた俗語が核にある
- 誇張表現: ありえない大きさ・量・強度で押している
- メタ的表現: 大喜利、回答、作者、AI、ネタであることに触れる
- パロディ: 既存作品・有名表現の型を借りている
- 言葉遊び: 音、表記、二重意味、言い換えが核にある
- 適度な長さ: 1読で理解でき、削りすぎでも長すぎでもない
- 過度に長い回答: 核に対して説明が多い
- 名詞比率が高すぎる: 名詞を並べるだけで動きや関係が弱い

---

## 採点

回答ごとに、各クラスタの「適合シグナル」を概算する。

1. 該当する好まれる特徴のスコアを加算する
2. 該当する嫌われる特徴のスコアを加算する
3. 弱い該当は半分で計算する
4. 表にない共通傾向は、強い場合のみ `+0.15` または `-0.15` として加える
5. 最終値は `-1.0` 〜 `+1.0` に丸める

点数は小数第2位まで。厳密な統計推定ではなく、開発用の近似である。

---

## 改善メモ

改善に使えるよう、結論は必ず次の形で書く。

- `広く取りに行くなら`: 複数クラスタに強い負点を作っている特徴をどう減らすか
- `尖らせるなら`: 最も適合しているクラスタに寄せるには何を足す・残すか
- `避けたい最適化`: 表だけをハックする危険がある変更

書いてよい例:
- `広く取りに行くなら、三点リーダーを落とす。C3/C4には効くがC6の負点が大きい。`
- `尖らせるなら、C0/C3向けに括弧内のズレを核にする。`
- `避けたい最適化: 括弧を足すだけではC0風の表層になり、回答の絵が弱くなる。`

書いてはいけない例:
- `C0の人はこういう性格なので笑う`
- `これは面白い`
- `つまらない`
- `万人受けする`

```
- 広く取りに行くなら: [具体的に]
- 尖らせるなら: [具体的に]
- 避けたい最適化: [具体的に]

（以下同様）

---

### 全体傾向
- 寄りが強いクラスタ: [クラスタと理由]
- 負点が出やすい特徴: [特徴]
- 回答群としての偏り: [同じ特徴への偏り、または「なし」]

### 使い方
この結果は面白さの絶対評価ではなく、嗜好特徴への適合シグナルである。
最終採用は `fun-check` のリスク検出や人間判断と併用する。
```

## 注意

- 点数を勝敗や順位に直結させるな
- クラスタを人格として説明するな
- 「面白い」「つまらない」という語で結論づけるな
- 特徴が見えない回答を無理に分類するな
- 改善メモは、必ず回答の文字面をどう変えるかに落とす

# 多様性チェック

大喜利の回答群を受け取り、各回答が「お題をどの軸で分解しているか」を分類する。

## 入力

お題と、それに対する回答群（5〜20個程度）を受け取る。

## 手順

### 1. 各回答の分解軸を特定

回答ごとに、**お題のどの要素を、どう読み替えているか**を一言で書け。

フォーマット：
```
回答: [回答テキスト]
軸: [お題のどの部分] → [どう読み替えたか]
```

注意：
- 「面白いか」は判断するな。軸の分類だけ行え
- 表面的な素材（葬式、電車、ゴルフ等）ではなく、お題に対するアプローチの構造を見ろ
- 同じ構造で素材が違うだけなら同じ軸

### 2. 軸の集約

同じ構造の軸をグループにまとめ、グループ名をつけろ。

### 3. レポート

```
## 分解軸レポート

お題: [お題]
回答数: N
軸の種類: M

### 軸一覧
- 軸A（N個）: [説明] — 回答1, 回答3, 回答7
- 軸B（N個）: [説明] — 回答2
- 軸C（N個）: [説明] — 回答4, 回答5
...

### 判定
[M ≥ 5: 十分 / 3-4: 改善の余地あり / 1-2: 収束している]
```

## 注意

- このスキルは回答の「面白さ」を評価しない
- 分解軸の種類数だけを測定する
- 開発時の指標として使う
